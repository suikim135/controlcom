# ControlCom 사용자 가이드

침대·소파에서 Wi-Fi로 내 PC를 조종하는 앱입니다.

---

## 1. ControlCom이 하는 일

| 버튼 | 동작 |
|------|------|
| 절전 모드 | PC를 Sleep 상태로 전환 |
| 음소거 | 시스템 음소거 ON/OFF |
| 모니터 1개만 | 주 모니터만 사용 / 듀얼 복구 |
| 저장 후 종료 | 열린 프로그램에 저장 시도 후 PC 종료 |

> **중요:** 폰 앱만으로는 PC를 제어할 수 없습니다. **PC에 ControlCom Agent**도 설치·실행되어 있어야 합니다.

---

## 2. 필요한 것

| 항목 | 요구사항 |
|------|----------|
| PC | Windows 10/11 |
| 스마트폰 | Android 8.0 이상 (API 26+) |
| 네트워크 | PC와 폰이 **같은 Wi-Fi** |
| 권한 | 폰: 인터넷, 카메라(QR 스캔 시) |

---

## 3. 전체 설치 흐름

```
① PC에 ControlCom Agent 설치·실행
        ↓
② 페어링 QR 화면 확인 (브라우저 자동 오픈)
        ↓
③ 폰에 ControlCom 앱 설치
        ↓
④ 앱 "시작하기" → QR 스캔 또는 IP+코드 입력
        ↓
⑤ 메인 화면에서 버튼 사용
```

---

## 4. PC 에이전트 설치

### 방법 A — 설치 파일 (권장)

1. `ControlComAgent-Setup.exe` 실행 (빌드 후 `pc-agent/installer/Output/` 에 생성)
2. 설치 마법사에서 다음 옵션 확인:
   - Windows 로그인 시 자동 실행
   - Private 네트워크 방화벽 허용 (포트 7847)
3. 설치 완료 후 브라우저에 **페어링 페이지**가 열립니다.

### 방법 B — 개발자 실행 (소스에서)

```powershell
cd pc-agent
dotnet run
```

최초 실행 전 [.NET 8 SDK](https://dotnet.microsoft.com/download) 필요.

### 방법 C — 수동 방화벽 (설치 파일 없이 실행 시)

관리자 PowerShell:

```powershell
cd pc-agent
.\scripts\add-firewall-rule.ps1
```

### 페어링 페이지 직접 열기

PC에서 브라우저 주소창에 입력:

```
http://localhost:7847/pair
```

화면에 **QR 코드**와 **6자리 코드**가 표시됩니다.

## 5. Android 앱 설치

### 방법 A — Play Store (출시 후)

Play Store에서 "ControlCom" 검색 후 설치.

### 방법 B — Snapdrop (APK 직접 설치, 권장)

카카오톡·구글 드라이브는 APK 업로드/다운로드를 **차단**하는 경우가 많습니다.

1. PC와 폰을 **같은 Wi-Fi**에 연결
2. 둘 다 브라우저에서 **https://snapdrop.net** 접속
3. PC에서 `dist/ControlCom-1.0.0-release.apk` 를 화면에 드래그
4. 폰에서 파일 수신 → **내 파일** 앱에서 APK 탭 → 설치

### 방법 C — PC Wi-Fi 서버

```powershell
cd scripts
.\install-to-phone.ps1
```

화면에 표시된 주소(예: `http://192.168.0.15:8765/`)를 폰 Chrome에서 열고 다운로드.

### 방법 D — USB + adb

```powershell
adb install dist\ControlCom-1.0.0-release.apk
```

### 설치가 막힐 때 (삼성 갤럭시)

**설정 → 보안 및 개인정보 보호 → 설치 차단 없는 앱**  
→ **내 파일** 또는 **Chrome** 허용

---

## 6. 첫 연결 (페어링)

앱을 처음 열면 **시작하기** 화면이 나옵니다.

### QR로 연결 (가장 쉬움)

1. PC 에이전트 실행 → 페어링 페이지(`http://localhost:7847/pair`) 확인
2. 폰 앱에서 **QR 코드로 페어링** 탭
3. PC 화면의 QR을 스캔
4. 자동으로 연결 완료 → 메인 화면 이동

### 수동으로 연결

1. PC 페어링 페이지에서 **IP 주소**와 **6자리 코드** 확인
2. 폰 앱에 PC IP (예: `192.168.0.15`), 포트 `7847`, 코드 입력
3. **연결하기** 탭

### PC IP 확인 방법

- 페어링 페이지에 표시됨
- 또는 PC에서 `ipconfig` → Wi-Fi 어댑터의 IPv4 주소

---

## 7. 앱 사용법

### 메인 화면

상단 **연결 상태** 표시:

| 표시 | 의미 |
|------|------|
| 연결됨 (녹색) | PC와 정상 통신 |
| 끊김 (빨간색) | Wi-Fi·에이전트·방화벽 확인 |
| 미연결 (회색) | IP 미설정 또는 미페어링 |

### 각 버튼

- **절전 모드** — 즉시 Sleep. PC가 잠들면 앱 응답 없음 (정상)
- **음소거** — 토글. 현재 상태가 버튼에 반영됨
- **모니터 1개만** — 주 모니터만 / 듀얼 복구 토글
- **저장 후 종료** — 확인 창 후 실행. **모든 앱 저장을 보장하지 않음**

### 설정 화면

- PC IP·포트 변경
- 연결 테스트
- QR 스캔 / 페어링 코드 재입력

## 8. 자주 묻는 질문

### PC 프로그램 없이 폰만으로 되나요?

안 됩니다. PC에서 명령을 실행할 **에이전트**가 반드시 필요합니다.

### 다른 집·외부에서 PC를 조종할 수 있나요?

아니요. **같은 Wi-Fi(LAN)** 에서만 동작합니다. 인터넷 원격 제어는 지원하지 않습니다.

### 카톡/구글 드라이브로 APK를 보낼 수 없어요

정상입니다. APK는 악성코드 방지 정책으로 **많은 서비스에서 차단**됩니다.  
→ [Snapdrop](https://snapdrop.net) 또는 USB/adb 사용.

### 페어링 코드가 안 맞아요

- PC 에이전트를 재시작하면 새 코드가 생성될 수 있음
- 이미 페어링된 PC면 코드 대신 앱에서 기존 토큰으로 연결
- PC 에이전트 데이터 삭제 후 재페어링:  
  `%LOCALAPPDATA%\ControlCom\Agent\` 폴더 삭제 후 에이전트 재실행

### 연결이 안 돼요

1. PC와 폰이 **같은 Wi-Fi**인지 확인
2. PC 에이전트가 **실행 중**인지 확인
3. PC IP가 맞는지 확인 (`ipconfig`)
4. Windows 방화벽에서 **포트 7847** 허용 확인
5. VPN이 켜져 있으면 끄고 재시도

### 저장 후 종료했는데 작업이 안 저장됐어요

일부 앱(Unity, Photoshop 등)은 Ctrl+S만으로 저장되지 않습니다.  
안전하게 끄려면 **절전 모드**를 사용하거나, PC 앞에서 직접 저장하세요.

---

## 9. PC 에이전트 자동 실행

상세: [상시 실행 가이드](GUIDE_ALWAYS_ON.md)

```powershell
cd pc-agent
.\scripts\install-startup-task.ps1
```

재부팅 후에도 로그인 시 자동 시작됩니다. 폰은 **재페어링 없이** 자동 재연결됩니다.

---

## 10. 앱 삭제·재설치

| 대상 | 방법 |
|------|------|
| Android 앱 | 일반 앱 삭제 |
| PC 에이전트 | 설정 → 앱 → ControlCom Agent 제거 |
| 페어링 초기화 | `%LOCALAPPDATA%\ControlCom\Agent\` 삭제 |

---

## 11. 보안 안내

- 통신은 **내 Wi-Fi 안에서만** 이루어집니다
- 외부 서버로 데이터를 보내지 않습니다
- 페어링 토큰은 PC·폰에만 저장됩니다
- 공유기 **포트포워딩은 하지 마세요**
