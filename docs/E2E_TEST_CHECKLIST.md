# ControlCom E2E 테스트 체크리스트

PC(Windows) + Android(S24+ 등) 실기기에서 **처음부터 끝까지** 검증할 때 사용하세요.  
순서대로 진행하며, 각 항목에 체크(✅)하세요.

**예상 소요:** 20~30분  
**필수 조건:** PC와 폰이 **같은 Wi-Fi**

---

## 0. 사전 준비

| # | 항목 | 확인 |
|---|------|------|
| 0-1 | PC: Windows 10/11, Wi-Fi 연결됨 | ☐ |
| 0-2 | 폰: Android 8.0+, Wi-Fi 연결됨 (PC와 **동일 네트워크**) | ☐ |
| 0-3 | PC에 .NET 8 SDK 설치됨 (`dotnet --version` 동작) | ☐ |
| 0-4 | 폰에 `dist/ControlCom-1.0.0-release.apk` 설치 준비 (Snapdrop 권장) | ☐ |

---

## 1. PC 에이전트 기동

```powershell
cd pc-agent
dotnet run
```

| # | 확인 내용 | 기대 결과 | OK |
|---|-----------|-----------|-----|
| 1-1 | 콘솔에 `Listening on http://0.0.0.0:7847` 표시 | 포트 7847 리스닝 | ☐ |
| 1-2 | 브라우저에 페어링 페이지 자동 오픈 | `http://localhost:7847/pair` | ☐ |
| 1-3 | 페어링 페이지에 QR + 6자리 코드 표시 | 코드 6자리 숫자 | ☐ |
| 1-4 | PC IP가 페이지에 표시됨 (예: 192.168.x.x) | 127.0.0.1이 아닌 LAN IP | ☐ |

### 1-A. 방화벽 (연결 실패 시)

관리자 PowerShell:

```powershell
cd pc-agent
.\scripts\add-firewall-rule.ps1
```

| # | 확인 | OK |
|---|------|-----|
| 1-A-1 | 방화벽 규칙 추가됨 (TCP 7847, Private) | ☐ |

### 1-B. PC 로컬 API 확인

PC 브라우저 또는 PowerShell:

```
http://localhost:7847/api/health
```

| # | 기대 응답 | OK |
|---|-----------|-----|
| 1-B-1 | `{"ok":true,"service":"controlcom-agent",...}` | ☐ |

## 2. Android 앱 설치

### Snapdrop (권장)

1. PC·폰 모두 https://snapdrop.net 접속
2. PC에서 `dist/ControlCom-1.0.0-release.apk` 드래그
3. 폰에서 수신 → **내 파일**에서 설치

| # | 확인 | OK |
|---|------|-----|
| 2-1 | APK 설치 완료 | ☐ |
| 2-2 | 앱 아이콘 "ControlCom" 실행됨 | ☐ |
| 2-3 | 첫 실행 시 **시작하기** 화면 표시 | ☐ |

> 카톡/드라이브 APK 차단은 정상. Snapdrop 또는 `scripts/install-to-phone.ps1` 사용.

## 3. 페어링

### 방법 A — QR (권장)

| # | 동작 | 기대 결과 | OK |
|---|------|-----------|-----|
| 3-A-1 | 앱 **QR 코드로 페어링** 탭 | 카메라 열림 | ☐ |
| 3-A-2 | PC 페어링 페이지 QR 스캔 | 스캔 성공 | ☐ |
| 3-A-3 | "QR 페어링이 완료되었습니다" 토스트 | 메시지 표시 | ☐ |
| 3-A-4 | 메인 화면으로 자동 이동 | 4개 버튼 보임 | ☐ |
| 3-A-5 | 상단 **연결됨** (녹색) | ConnectionStatus.Connected | ☐ |

### 방법 B — 수동 (QR 실패 시)

| # | 동작 | 기대 결과 | OK |
|---|------|-----------|-----|
| 3-B-1 | PC IP, 포트 `7847`, 6자리 코드 입력 | — | ☐ |
| 3-B-2 | **연결하기** 탭 | 페어링 성공 메시지 | ☐ |
| 3-B-3 | 메인 화면, **연결됨** 표시 | ☐ | ☐ |

### 페어링 실패 시 점검

| 원인 | 조치 |
|------|------|
| 연결 끊김 | PC IP 재확인 (`ipconfig`), 같은 Wi-Fi 확인 |
| 401 / 페어링 필요 | PC 에이전트 재시작 → 새 코드로 재시도 |
| 이미 페어링됨 | `%LOCALAPPDATA%\ControlCom\Agent\` 삭제 후 에이전트 재실행 |

## 4. 핵심 기능 (PC 앞에서 눈으로 확인)

> **주의:** Sleep/종료는 실제 PC 상태가 바뀝니다. 저장 안 된 작업 있으면 먼저 저장하세요.

### 4-1. 음소거 (가장 안전, 먼저 테스트)

| # | 동작 | PC에서 확인 | OK |
|---|------|-------------|-----|
| 4-1-1 | PC에서 음악/유튜브 재생 | 소리 남 | ☐ |
| 4-1-2 | 앱 **음소거** 탭 | 소리 꺼짐 | ☐ |
| 4-1-3 | 버튼이 **음소거 해제**로 바뀜 | UI 반영 | ☐ |
| 4-1-4 | 다시 탭 | 소리 복구 | ☐ |

### 4-2. 모니터 1개 / 듀얼

| # | 동작 | PC에서 확인 | OK |
|---|------|-------------|-----|
| 4-2-1 | **모니터 1개만** 탭 | 보조 모니터 꺼짐 (주 모니터만) | ☐ |
| 4-2-2 | 버튼이 **듀얼 모니터 복구**로 바뀜 | UI 반영 | ☐ |
| 4-2-3 | 다시 탭 | 확장(듀얼) 복구 | ☐ |

> 듀얼 모니터가 없으면 이 항목은 스킵. 단일 모니터에서는 변화 없을 수 있음.

### 4-3. 절전 모드 (Sleep)

| # | 동작 | PC에서 확인 | OK |
|---|------|-------------|-----|
| 4-3-1 | 앱 **절전 모드** 탭 | PC Sleep 진입 | ☐ |
| 4-3-2 | 앱 "절전 명령을 전송했습니다" 토스트 | ☐ |
| 4-3-3 | PC 전원 버튼/키보드로 깨우기 | 정상 복귀 | ☐ |
| 4-3-4 | 깨운 후 앱 상단 **연결됨** 복구 (수 초 내) | ☐ |

### 4-4. 저장 후 종료 (마지막에 테스트)

**테스트 전:** 메모장에 아무 글 쓰고 **저장 안 한 상태**로 두기 (동작 확인용).

| # | 동작 | PC에서 확인 | OK |
|---|------|-------------|-----|
| 4-4-1 | **저장 후 종료** → 확인 다이얼로그 **종료** | — | ☐ |
| 4-4-2 | 메모장 등에 저장 대화상자 또는 저장 시도 | 최선 저장 동작 | ☐ |
| 4-4-3 | 약 10초 후 PC 종료 예약/종료 | `shutdown /s /t 10` | ☐ |

> 종료 취소: PC에서 `shutdown /a` (관리자 CMD)
